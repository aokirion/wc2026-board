# CLAUDE.md — 2026 W杯 生存ボード 自動更新タスク

毎日 9:00 / 17:00（JST）を目安に**手動で起動**し、最新スコアを反映する（自動スケジュール実行ではない。ユーザーがプロンプトを貼って起動する運用）。
**モデルは Sonnet 固定。** API キーは使わず、サブスク（Pro/Max）認証で動かすこと。

## 絶対ルール
- 編集してよいのは **`data.json` だけ**。`index.html` / `.claude/` / `scripts/` などは絶対に触らない（フックでもブロック済み）。
- 反映するスコアは **「試合終了（Full-time / Final）」が確定したものだけ**。試合中・未開催・速報の暫定値は入れない。
- スコアは **FIFA公式サイト（fifa.com）を一次ソースとして必ず確認する**。FIFA公式の試合ページ（`fifa.com/en/match-centre/...`）にスコアが掲載されていることを確認してから反映する。**推測でスコアを入れない。他ソースのみでは反映しない。**
- 突破（advanced）・敗退（eliminated）は **公式に確定した場合のみ** 反映。不確実なら `alive` のまま。
- **スコアだけでなく、毎回「突破・敗退・グループ最終順位」も更新対象に含める**（下記手順3参照）。新しいスコアが無くても status/rank の確定があれば反映する。
- チーム名（キー）は既存の表記を一字一句そのまま使う。新しい綴りを作らない。

## 毎回の手順
1. `data.json` を読む。`"played": false` の試合を洗い出す。
2. 各試合について **FIFA公式サイト（fifa.com/en/match-centre/...）** を検索・確認し、**"Full-time" 表示のある試合のみスコアを取得**する。
3. `data.json` を編集する：
   - 終了した試合：`"hs"` `"as"`（整数）を入れ `"played": true` にする。
   - 既に終わっているのに一覧に無い試合（次節・ノックアウト等）：matches に追加してよい。
     - グループ戦は `"g"` にグループ記号（A〜L）。ノックアウトは `"g":"R32"` / `"R16"` / `"QF"` / `"SF"` / `"3rd"` / `"FIN"`（決勝）。
       - **決勝は必ず `"FIN"`。`"F"` はグループFと衝突するので使わない。**
       - ノックアウトの試合を追加すると `index.html` の「決勝T」タブにブラケット表示される。
       - 実カード追加前は、確定した順位（rank 1/2）から公式R32組み合わせを自動充填した「スケルトン」が表示される。全グループ終了後にFIFA公式でR32対戦カードが確定したら、`matches` に `g:"R32"` で実カードを追加するとスコア付きブラケットに切り替わる。
       - **PK決着**（90分/延長が同点でPK戦）の場合は、`hs`/`as` に90分(or延長)終了時の同点スコアを入れ、`"pkw":"h"`（ホーム勝ち）または `"pkw":"a"`（アウェイ勝ち）を付ける。これでブラケットの勝者ハイライトが正しく出る。
     - 未開催なら `"played": false`、`"d"` に日付（例 `"6/19"`）。
   - 突破/敗退が公式確定したチーム：`teams` の該当国の `"status"` を `"advanced"`（突破）または `"eliminated"`（敗退）に。FIFA公式（試合レポート、または `fifa.com/.../articles/qualified-eliminated-teams`）で確定を確認すること。
   - **グループ最終順位が確定したチーム：`teams` の該当国に `"rank": 1`（1位）/`2`（2位）/`3`（3位）を付ける。**
     - **数学的 or 公式に順位が確定した場合のみ**。1位/2位が未確定（例：勝点・得失点で並びうる）のうちは付けない。確定したものから順次。
     - rank は「決勝T」タブの進出チームに順位バッジ（1位=金/2位=銀/3位=銅）として表示される。
     - 敗退国に順位は不要（付けない）。
   - `meta.asOf` を本日（JST、例 `"2026/6/14"`）に更新。
4. **検証：** `node scripts/validate.mjs` を実行。
   - 失敗したら `git checkout -- data.json` で戻し、**push せずに終了**（壊れたデータを公開しない）。
5. **公開：** `git status` で `data.json` に変更があれば、`git add data.json`（**data.json のみ**）→ `git commit` → `git push`。
   - 変更が無ければ何もしない（コミット不要）。

## メモ
- 冪等：同じ試合を二重に入れない。既に正しいスコアが入っていれば変更しない。
- 1試合でも確定していれば反映する（全部揃うのを待たない）。
- 迷ったら **保守的に**（入れない・status を変えない）。手動で `data.json` を直せば上書きできる。
